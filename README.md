# Raspberry Pi Smart Light System

## Overview

This project implements an automatic lighting system using a Raspberry Pi, a photoresistor (LDR), and a PCF8591 analog-to-digital converter.

The system continuously monitors ambient light levels and automatically adjusts the brightness of an LED using PWM (Pulse Width Modulation). As the environment becomes darker, the LED brightness increases; as ambient light increases, the LED dims.

The project was developed to explore analog sensing, ADCs, PWM control, and basic feedback systems in embedded applications.

## How It Works

1. The photoresistor changes resistance according to ambient light.
2. A voltage divider converts the resistance change into a varying voltage.
3. The PCF8591 measures this analog voltage and converts it into a digital value.
4. The Raspberry Pi reads the converted value over the I²C bus.
5. The sensor reading is mapped to a PWM duty cycle.
6. The LED brightness is adjusted automatically in response to environmental lighting conditions.

---

## Calibration Considerations

The light sensor values used in this project were obtained experimentally under the lighting conditions of my workspace.

Because ambient lighting varies significantly between environments, the minimum and maximum sensor values used for mapping should be recalibrated when the system is deployed in a different location. This highlights an important aspect of real-world sensor systems: calibration is often required to achieve reliable performance.

## Concepts Demonstrated

### Analog-to-Digital Conversion (ADC)

The Raspberry Pi cannot directly measure analog voltages. The PCF8591 converts the analog voltage generated by the photoresistor circuit into digital values that can be processed in software.

### I²C Communication

Sensor data is transmitted between the Raspberry Pi and the PCF8591 using the I²C communication protocol.

### PWM Control

LED brightness is controlled through Pulse Width Modulation (PWM), allowing smooth intensity adjustment without changing the supply voltage.

### Feedback Systems

The project implements a basic feedback loop:

Ambient Light → Sensor → Processing → LED Output

This introduces fundamental concepts used in robotics, automation, and control systems.

## Design Evolution

The initial version of the project focused on controlling the brightness of a white LED based on ambient light conditions. Once this was working, I explored a more challenging idea: extending the same behavior to an RGB LED while preserving the selected color.

This introduced an additional problem. Simply mapping the ambient light value directly to the RGB channels affects the balance between the red, green, and blue components, which can alter the original color instead of only changing its brightness.

My current implementation works but is not fully optimized. While it successfully adapts brightness to environmental conditions, it can slightly distort the intended color but without actually affecting the RGB balance. I am currently investigating alternative approaches that would preserve the original color ratios while scaling overall intensity more efficiently.

## What I Learned

## Lessons Learned

One of the most surprising discoveries during this project was learning that the Raspberry Pi cannot directly measure analog voltages through its GPIO pins.

Coming from an Arduino background, where analog inputs are available through built-in ADC hardware, I initially assumed the Raspberry Pi offered similar functionality. Through this project, I learned that analog sensors require an external Analog-to-Digital Converter (ADC) such as the PCF8591 before their measurements can be processed by the Raspberry Pi.

Through this project, I gained practical experience in integrating analog sensors with digital systems. I learned how analog physical phenomena can be converted into digital data through ADCs, how devices communicate over an I²C bus, and how PWM can be used to control hardware outputs.

The project also introduced me to feedback control concepts, where sensor measurements are continuously used to adjust system behavior in real time. These principles are widely used in embedded systems, robotics, industrial automation, and IoT applications.

The demonstration was performed using a small enclosed box to create a controlled lighting environment. The photoresistor was placed inside the box while the RGB LED remained visible outside.

To simulate a sudden increase in ambient light, the flashlight of a smartphone was directed into the box. As the light sensor detected the increased illumination, the system automatically reduced the LED brightness. When the light source was removed, the LED returned to a higher brightness level.

This demonstrates the complete feedback loop implemented by the project:

Ambient Light → Photoresistor → PCF8591 ADC → Raspberry Pi → PWM Output → LED Brightness

The test also highlights the system's ability to respond dynamically to changing environmental conditions without user intervention.
